# FAQ

**Do you work with existing codebases, or only from scratch?**

Both. I enjoy greenfield work, but I'm comfortable dropping into an existing codebase, understanding it, and working within it.

**Do you take long-term contracts or short project-based work?**

Either, though right now I lean toward shorter, well-scoped tasks.

**Do you work solo or as part of a team?**

Solo, currently.

**Do you handle deployment and infrastructure, or just write the code?**

All of it — I plan, build and deploy myself. Backend, infrastructure, DevOps, and frontend when the project needs it.

**How do you usually communicate during a project?**

Mostly text, on Telegram. Calls work too if that's easier for a given conversation.

**Do you write documentation?**

I comment my code as I go. Full documentation isn't a default deliverable, but I'll write it if the project calls for it.
